Installation and Usage

Installation using UX command line utility (recommended)

Requirements

  • NodeJS
  • Bower (npm install bower -g)
  • UX-CLI (npm install @unumux/ux-cli -g)

UX-CLI usage

UX-CLI is a command line utility used to run our build tools, and also to scaffold out static marketing sites.

UX-CLI can be used by opening your terminal or command prompt, and using cd to navigate to your project directory. Running ux will ask a series of questions to set up a new project, or run an existing one based on a ux.json file. More information on ux-cli can be read here

Using Colonial Branding

Colonial branding (the framework items shown above) can be installed via the UX tool during project initialization, or via bower as listed below. If you want to use the UX tool to install libraries after initialization, you can use the command ux --install

The branding files will also have to be imported into your SCSS stylesheet. This can be done by putting @import "colonial-branding/main"; in your main SCSS file.

Installation using Bower

If you want to use the branding files outside of our UX build process, you can install using bower.

Requirements

  • NodeJS
  • Bower (npm install bower -g)

The bower install can be done by typing bower install unumux/colonial-branding --save

Download of CSS stylesheets (unrecommended)

The framework can also be obtained by downloading the CSS stylesheets. These will have to be added to your markup using a link tag. This method is not recommended as it will not optimize the framework styles with the custom site styles, but it is made available for special circumstances.

The file main.css is the file you need, unless you also want to use the bootstrap JS for various components. The "lite.css" includes less CSS, and should not be used at the same time as "main.css". The "colonial-branding.zip" includes everything
